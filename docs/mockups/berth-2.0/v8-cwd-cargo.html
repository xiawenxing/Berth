项目默认目录：Berth 自动分配的工作区（物理上是 ~/.berth/workspaces/<项目>/），起航没选 cwd 的会话落这儿。分组里**只显示「项目默认目录」**，不露真实磁盘路径。它**不进货舱登记**，也没有「导入」icon。
其余组：真实 cwd（货舱里登记的目录 / 会话实际跑过的目录），打码用 ~ 缩写。每组右侧 导入 icon = 把该目录下磁盘上、尚未导入的其他会话拉进来（会话粒度勾选）。

真实目录，写进项目的 project_path；无 ★ 主标记——只保留 开关（沿用现有 <Toggle>）：开 = 起航默认把它当 cwd 候选，关 = 登记备用、不进起航默认。
「项目默认目录」不出现在这里——它是 Berth 内部分配、不可登记的兜底。新建项目时也能填一个主目录（留空就只用项目默认目录）。

3 起航 — 启动 cwd 怎么定（无 ★，自动点亮一个、可切换）

起航

启动目录
项目默认目录 自动 · 无装载目录
没有开启任何货舱 → 落 Berth 项目默认工作区。无需选择。
~/Code/berth 自动 · 唯一装载目录
只开启了 1 个货舱 → 直接在它下启动。无需选择。
开启了多个货舱 → 自动点亮一个（沿用上次该项目的启动 cwd，否则第一个）；可手动切到别的，本次生效并记住为下次默认。

解析顺序：本次选中的装载 cwd → 唯一装载则取它 → 都没有则落「项目默认目录」。自动点亮规则：该项目上次启动的 cwd（粘性），无记录则第一个开启的。

两个入口、两种语义：
① 货舱「添加目录」 → 登记目录是主操作（写 project_path enabled=1），导入会话可选；按钮：选 0 = 仅登记目录，选 N = 登记并导入 (N)。
② 会话模块某 cwd 组的导入 icon（该目录已登记）→ 纯导入；按钮：导入选中 (N)，选 0 时禁用。
导入 = 选中会话 addSessionImport（+项目入口则 attach）；未选留在「无归属」，目录照样登记成功。
