项目小结 / 任务进展详情 — 小标题样式选项

需求：小标题（一句话总结 / 进度要点 / 重要里程碑）前不再用圆点，避免与正文圆点混淆。下面 4 个方案 + 现状参考，挑一个我直接落地。

现状（参考）

小标题前是彩色圆点，和正文要点的蓝圆点同形，容易混。

A · 彩色文字标签（推荐）

去掉圆点，标题文字用区分色（橙/绿/蓝）+ 更粗字重。最简洁，无任何前缀符号与圆点冲突。

B · 左侧竖条

标题前用一根彩色短竖条代替圆点，文字保持灰。竖条≠圆点，形状上就分开了。

C · 图标前缀

每个区配一个语义图标（✦ 总结 / ☰ 进度 / ✓ 里程碑）。最直观，但视觉比圆点重。

D · 细线标题（无色）

完全去色，标题下加一条细分隔线分区。最克制、最“文档感”，但失去颜色区分。
