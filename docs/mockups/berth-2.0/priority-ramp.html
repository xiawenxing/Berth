优先级色带 · 任意档位自动配色（日间）

用户在设置页增减优先级 → 在有序色带上等距采样 N 个点。每条策略左 5 档、右 7 档：看越靠上越热/越重、相邻档能否分清、整体是否和谐。bar=种子色，chip 底/字按固定明度派生（任意档都 ≥AA）。颜色由 oklch 算法生成，不是手填、也不是随机。

升温·经紫 推荐
种子色从绛红(最高)平滑过到午夜靛蓝(最低)，色相走暖红→莓→紫→靛的短路径（不经浑浊的黄绿），彩度集中在高优端、低优端近灰。最像「日落→午夜」，多等级也清晰有序、与品牌靛蓝收尾。

顶部独热
只有最高优是暖绛红，其余全部是「冷靛蓝→淡」的灰阶。最贴合「只有紧急的有色、其它都安静」——当只有 1 档告警 + 多档常态时最干净。

暖→中性灰
色相从警示红渐移到中性、彩度降到近 0，最低优是纯中性灰（不走蓝）。适合不想让「低优=蓝色」被误读成另一类信息的场景。

分配规则：优先级是有序列表，第 i 个（i 从 0=最高）取色带位置 t=i/(N-1)。同一优先级永远同色（确定性，不随机、不闪烁）。
我的排序：① 升温·经紫（多档最稳、收尾靛蓝最贴品牌）② 暖→中性灰（不引入蓝、最克制）③ 单色强度（同色家族最协调，但多档靠深浅分）。顶部独热只适合「单一告警档」。
