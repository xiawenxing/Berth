## 0. 背景与核心理念

同事试用反馈「**不知道核心功能是什么、不会用**」。审计确认根因两层：① 没有"主对象"——导航以时间流 Now 打头，项目/全部会话是同一批会话的两种分组，并列摆着，从不点名核心对象；② 同一批会话被 3–4 套渲染器在多个界面重复（会话行 3 套、项目列表 3 套、起会话入口 5+），信号被稀释。

**核心理念（权威）**：Berth 管的是 **项目与任务（交付物）**，用 agent 去执行它们。一个会话是**次抛的执行工具**——一艘**船**，装载**货舱（上下文）**离港、出航、某天带着结果**返港**。Berth = 帮你经营这个**港湾**，提高出货/进货量。**任务领衔，会话服务**。会话的价值是它**交付的结果**或**沉淀**下来的知识，不是每次去 resume 的 transcript。

## 1. 信息架构（IA）

- **导航三项**：`Now` / `项目` / `设置`。**废弃**「全部会话」页与「项目」导览卡片页。
- **左栏 = 项目切换器**：`+ 新建项目` · 置顶项目 · 活跃项目 · 已归档项目（折叠）· `无归属会话` 入口。点「项目」**直接进入某项目工作台**（无中间卡片墙）。
- **四个页面**：项目工作台（hub）、Now（跨项目收件箱）、无归属会话（扁平列表）、设置。
- **被废「全部会话」的三件独占工作已安置**：无归属桶 + 导入目录 → 无归属会话页 / 设置；全局会话搜索 → 无归属页（后续可加全局命令面板）。

## 2. 项目工作台（hub）—— 任务优先布局

自上而下单列滚动：

1. **粘性头部**：项目名 + path + rollup（任务进展 N/M · 进行中 · 最近活动）+ **港湾概览**药丸（在跑 / 靠岸·待查收 / 今日交付 N/M）+ 动作（`新建任务` 主按钮 / `起会话` / `小结` / `⋯`）。
2. **任务（英雄区，标签「航线」）**：看板列 待办/进行中/待评估/已完成/已取消。
   - **激活列加宽**：点列头激活该列（flex-grow 2.2），其余收窄成紧凑卡；**默认激活「进行中」**。
   - **限高 ~700px** 内部纵向滚动，列头 sticky。
   - **卡片规范**：13px/600 标题；优先级 = **4px 左色条**（P0红/P1琥珀/P2淡，非芯片）；`▷启动` 在**标题栏**且仅收起态显示（展开态有起会话按钮）；安静脚注：link N · 状态 glyph · **DDL** · `⋯`。点击展开 → **进展摘要 + 关联会话** mini 行。
   - **DDL**：文字式，未设→淡「截止」；设了→`今日`(琥珀)/日期/`逾期`(红)，点击弹快捷设置；`今日` DDL 喂「今日交付」计数。
   - 优先级改：点左色条/`⋯` 菜单；状态改：列间**拖拽** + `⋯` 菜单；0 关联会话展开 → 只给「起会话」按钮。
3. **会话（标签「船只」，次级模块）**：`📌 Pin` + **按 cwd 分组**（无 活跃/已归档）。**已停泊（闲置）行不显状态圆点**；只 在航=绿点 / 靠岸·待查收=蓝环。行：状态 glyph + CLI badge + 标题 + 时间 + hover 动作（pin/关联任务/重命名/`⋯`[移出项目, 停止会话]）；长 cwd 组 show more/less；已关联任务的行带 `已关联任务` 去重标。点击行 → **会话抽屉**。
4. **默认装载**：两个 UI 一致的登记表区块——`上下文文档`（项目+任务上下文，行可点开编辑）+ `代码上下文`（cwd/worktree 登记 + `默认装载` 开关）。

**词汇**：港湾单一方言 `在航 / 靠岸·待查收 / 已停泊`，Berth chrome 内不混用 "running"。

## 3. 起会话 —— 统一「装载台 / 起航」

所有 `▷起会话`（头部 / 会话模块 / 任务卡）打开**同一个装载台**：

- **目的地**：`任务`（从任务卡来时预选）| `自由提问`。
- **货舱（一个合并 block）**：摘要行即块头 + `调整装载` 就地展开编辑器（`☑项目上下文` `☑任务上下文` + `代码上下文`：登记的 cwd/worktree（默认装载项预勾）+ `额外目录…`(→`--add-dir`) + `CLI/模型`）。**开箱即走**（默认收起摘要 + `起航`）。
- `起航` → 打开**会话抽屉**。

货舱 = 现有 manifest 通道（项目 md / 任务 md 静默注入）+ 启动 cwd + `--add-dir` 的 UI 收口。

## 4. 会话抽屉（会话内容）

- **右侧 60vw 抽屉**，左侧页面变暗；关闭 = 点左暗区 / Esc（**头部无 ■停止 / ×关闭**）。
- **slim 头**：CLI badge · 标题 · cwd · 状态。
- **正文 = 对话式**（codex 风）：用户消息靠右气泡；agent 消息靠左富文本（行内 code + 代码块带复制 + **执行过程可折叠**「用时 Xs ▸」+ 文件改动卡片）。
- **底部输入框** + **一个按钮**：默认 `发送 ▷`，会话**在航时变 `终止 ■`**（destructive）。

## 5. 港务助手（AI）—— 交互原则

**核心原则：不让你等待。** 你的动作**即时生效**；AI 在**后台**把它做得更好，**原地更新**（不要 skeleton 等待、不要后台通知再回来查收）。只有"判断类"的帮助（关联/进度）作为**建议**让你确认。加载图标 = **sparkle 闪烁**（`spkTwinkle`）。

- **新建任务**：一个标题文本域 + `☑AI 自动总结任务标题` + `☐立即执行`。创建 = **即时建卡**（待办/进行中），后台 ~2s 把标题改精炼、把描述沉淀成进展摘要（卡上 sparkle「总结中…」）；`立即执行` → 同时起航并开抽屉。去掉优先级/状态/截止选择（建好后在卡上改）。
- **新建项目**：名称 + 描述 + `☑AI 根据描述生成项目上下文`。**不绑定主仓库路径**（路径在货舱指定）。创建即时进左栏；勾选则后台生成项目上下文（sparkle「生成上下文中…」）。
- **上下文更新（三路径）**：① 手动编辑文档；② 编辑面板底部输入框「**写一句 → 让 AI 整理更新**」（AI 整理并合并进上下文，sparkle「整理中…」）；③ 会话在该项目/任务下运行/返港时进展自动追加（被动）。**移除**「据最近会话刷新」按钮、上下文行 hover ↻、会话行/项目菜单的「刷新上下文」。
- **项目小结**：头部「小结」→ 面板，AI 生成（sparkle「生成中…」）当前进度 / 本周进展 / 下一步 → `应用为项目小结` / 复制。
- **C 建议（提议不应用）**：**触发条件** = 一艘船**返港**（在航→停泊）且**有实质产出**（文件改动/commit/推送/总结性结尾）且（**未归属任务** 或 **归属任务进度久未更新**）→ 给 **1 张**建议卡（关联航线+记进展 / 据产出建任务），`采纳`/`忽略`。**不触发**：无产出 / scratch 临时会话（`~/.berth/scratch`、`~/Downloads`）/ 已处理或已忽略过的。全局可关「主动提议」。
- **会话标题**：即时生成 + 一键撤销。

> 已**否决**的交互：A 内联草稿（skeleton 等待填表）、B 后台 job + 通知 + 流式查收 —— 都让用户等待。

## 6. Now（跨项目收件箱）

- 内容**撑满宽度**。**今日交付**任务置顶（跨项目，ddl=今日，可展开看进展摘要+关联会话，行带 `▷起会话` → 起航）。
- 下面 **船只区域**，按 `📌 Pin / 未读(靠岸·待查收) / 运行中(在航)` 三档，每行带**项目标签** + CLI badge，复用标准 34px 列表行。
- **无**港湾概览条、**无**「最近」。

## 7. 无归属会话

- 左栏入口 → **1.0 列表式两栏**：左侧常驻会话列表（**按 cwd 可折叠** + 组内 **show more/less**，已停泊无圆点，标准 34px 行，顶部搜索 + 导入目录，行动作含 `归属到项目`）+ 右侧会话内容（slim 头无 ■/×、对话式正文、底部输入 + 发送/终止）。点行**就地切换**右侧、不跳走。

## 8. 设置

六块设置卡：`外观`（主题 + 配色方案 swatches，午夜靛蓝默认）/ `港务助手`（管理模型 + 主动提议 + 默认 AI 总结标题）/ `启动 Agents`（claude/codex/coco 启用 + 默认模型；coco 无 `--model`）/ `上下文与文档`（docsRoot + 会话导入目录）/ `数据源·同步`（Feishu 多维表格 拉取/推送）/ `任务字段`（状态/优先级可编辑）。

## 9. 视觉系统（午夜靛蓝 + UI 约定）

**令牌**：`--background:#141b2e · --card:#1f2940 · --popover:#283450 · --border:#3a4768 · --sidebar:#101526 · --canvas:#0d1220 · --brand:#56b6ff · --success:#7ed4a6 · --warning:#f0c773 · --priority:#e0a060 · --purple:#b88cf0 · --destructive:#f0707f`（亮色保留可切换）。

**约定**：字号 模块/标题 13px/600 · 正文 13px · 次要 11px · chip 10.5px · 页 H1 16–17px。**强调** = 字重 600 + 前景色 + 可选 4px 左色条/小点（不靠放大字号/大留白）。**弱化** = muted + 11px + hover 才显动作。**标准列表行** ~34px：`[状态glyph][CLI badge][标题 13px][右:时间]` + hover 动作 + 选中=sidebar-accent+3px brand 左条。**紧凑卡片** + **登记表区块**统一。CLI：claude=brand / codex=success / coco=purple。

## 10. 数据模型 / 后端改动点

- **不引入会话归档**（会话只有 pin + cwd 分组；活跃/已归档被否决）。
- `project_path`（home cwd + added paths）**前台化**为「代码上下文」登记表 + 每条 `默认装载` 位 → 起航时映射到启动 cwd / `--add-dir`。**项目不再强绑单一仓库路径**。
- `task_ddl`（本地 deadline overlay）驱动「今日交付」。
- **复用管理 agent**（`runAgent`/`generateTitle`/`generateProgressSummary`/`consolidateContext`）：任务标题据描述总结、项目上下文据描述生成、上下文据输入 AI 整理合并、项目小结、会话标题。**新增**：C 建议生成（会话返港触发）。
- 起会话：货舱选择映射到现有 manifest（项目/任务 md 注入）+ cwd + `--add-dir`。
- 路由：删 `#/sessions`（全部会话）；无归属/导入/搜索 改由无归属页 + 设置承接。

## 11. 实现计划（分阶段，目标 = 对齐 `v7-project.html`）

> 全部在 `release/berth-2.0-ia` 分支；每阶段 `npx tsc --noEmit` clean + `npm test` green 后 commit；每阶段对照 mockup 查差距、迭代。

- **P0 路由/导航**：删「全部会话」与「项目」导览页；`项目` 直接进工作台；左栏 = 项目切换器（+ 新建项目）；午夜靛蓝令牌落地（`tokens.css`）。
- **P1 项目工作台布局**：任务优先（看板激活列 + 700 限高 + 卡片规范：左色条/DDL/▷启动/展开进展摘要+关联会话）；会话模块 Pin+cwd + 已停泊无点 + show-more + 已关联任务标。
- **P2 装载台/货舱**：用统一装载台替换 `openLaunchPopover`；默认装载模块（project_path 登记 + 默认装载位）。
- **P3 会话抽屉**：60vw 右抽屉 + 对话式正文 + 输入 发送/终止，替换现终端视图切换。
- **P4 即时创建**：新建任务（即时建卡 + 后台总结标题/补描述 + 立即执行）；新建项目（+ AI 生成项目上下文）。
- **P5 上下文更新**：上下文编辑面板 + 「写一句→AI 整理更新」输入；移除据会话刷新各处。
- **P6 其余页面与 AI**：Now 重构；无归属页；设置页；项目小结；C 建议（触发条件）；sparkle 加载。
- **R 收尾**：逐页对照 mockup 差距清单 → 迭代直到完全实现。
